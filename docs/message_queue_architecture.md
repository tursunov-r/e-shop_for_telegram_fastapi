# Архитектура асинхронной обработки заказов для SFMShop

## Задачи для очереди
- **Отправка email-уведомлений**  
  Подтверждение оформления заказа, уведомления о статусе доставки, маркетинговые рассылки.
- **Обновление склада**  
  Асинхронное уменьшение остатков после покупки, синхронизация с внешними системами учёта.
- **Генерация отчетов**  
  Формирование статистики продаж, аналитика по заказам и клиентам, периодическая агрегация данных.
- **Другие фоновые задачи**  
  Логирование действий пользователей, очистка устаревших данных, интеграции с внешними API (например, службы доставки).

## Надежность
- **Гарантии доставки сообщений**  
  Использование подтверждений доставки (ACK) в RabbitMQ. Продюсер получает подтверждение, что сообщение принято брокером, а консьюмер подтверждает успешную обработку.
- **Retry-логика при ошибках**  
  Повторная отправка сообщений при сбоях. Использование отложенных очередей или механизмов повторной доставки с экспоненциальной задержкой.
- **Обработка ошибок**  
  Dead-letter queue (DLQ) для хранения сообщений, которые не удалось обработать после нескольких попыток. Логирование ошибок и уведомление разработчиков.

## Масштабирование
- **Горизонтальное масштабирование Consumer**  
  Запуск нескольких воркеров (консьюмеров) для параллельной обработки задач. Использование контейнеризации (Docker) и оркестраторов (Kubernetes).
- **Балансировка нагрузки между воркерами**  
  RabbitMQ автоматически распределяет сообщения между активными консьюмерами очереди. Настройка `prefetch count` позволяет управлять количеством сообщений на воркера.
- **Мониторинг очередей**  
  RabbitMQ Management Plugin для визуального контроля. Интеграция с Prometheus и Grafana для сбора метрик и построения дашбордов. Настройка алертов на задержки, переполнение очередей и ошибки обработки.

## Итог
RabbitMQ обеспечивает **надёжную доставку сообщений** и гибкое масштабирование. Email-уведомления и обновление склада выполняются асинхронно, что снижает нагрузку на основной API и ускоряет отклик для пользователя. Архитектура легко расширяется и поддерживает высокую нагрузку, что критически важно для e-commerce проектов вроде SFMShop.
