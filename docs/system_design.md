# Системный дизайн проекта SFMShop

## Архитектура

### Компоненты:
- **Load Balancer** — распределяет входящие запросы, выполняет SSL‑терминацию и health checks.
- **API серверы** — несколько экземпляров, stateless‑архитектура для простого горизонтального масштабирования.
- **Redis (кэш)** — хранение сессий, rate limiting, query cache и full‑page cache.
- **PostgreSQL (БД)** — основной узел (master) + реплики для чтения, автоматический failover, резервное копирование.

### Масштабирование:
- **API** — горизонтальное масштабирование.
- **База данных** — вертикальное масштабирование основного узла, добавление реплик для чтения, возможное шардирование.
- **Очереди и кэш** — масштабирование Redis для обработки пиковых нагрузок.

### Кэширование:
- Redis для хранения часто используемых данных и результатов запросов.
- TTL для устаревающих данных.
- Кэширование страниц и статических ресурсов через CDN.

## Метрики производительности
- **RPS**: 5000+
- **Latency (p95)**: <200 мс
- **Availability**: 99.99%
- **Error rate**: <0.1%

### План масштабирования при росте нагрузки:
- Краткосрочно — авто‑масштабирование API и Redis.
- Среднесрочно — добавление реплик PostgreSQL.
- Долгосрочно — шардирование БД, переход к микросервисной архитектуре, мульти‑региональное развертывание.
